##  最好需要什么

1. 先理解理解完架构了，读起来就比较快了。
2. 学会使用它，不会使用，永远无法理解它的设计理念。
3. 读源码还得带着问题去读，不然会很枯燥。一定要带着问题去读，不然就会很枯燥，然后陷入细节中去
4. 读源码还得有一定积累量，比如你已经读过很多基础包的源码比如 net/http grpc-go 等等
5. 要尝试着调试，要尝试着练习，要尝试着理解和分析产品理念，更要学会自己设计。
